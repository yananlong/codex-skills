# Research Novelty Review

## Quick start

1. State the claimed contribution in one sentence.
2. Separate novelty of problem framing, method, evaluation protocol, artifact, and empirical finding.
3. Build a prior-art matrix before making novelty claims.
4. Pressure-test the strongest “this is incremental” objection.

## Workflow

### 1) Disaggregate the novelty claim

- Split claims into:
  - new question or framing
  - new method
  - new evaluation protocol
  - new benchmark or dataset
  - new empirical finding
- Reject blended novelty statements that hide which part is actually new.

### 2) Build the comparison set

- Identify the closest prior works, not a flattering long tail.
- For each comparison, record what overlaps and what is meaningfully different.
- Prefer matrices and concrete dimensions over narrative hand-waving.

### 3) Red-team the contribution

- Ask what a skeptical reviewer would call incremental.
- Flag claims that rely on renaming known patterns or bundling standard pieces.
- Distinguish “useful combination” from “methodological novelty.”

### 4) Tighten the positioning

- Narrow the claim until it is defensible.
- Prefer “first explicit X for Y” only when you can support it.
- Convert weak novelty claims into stronger value claims when appropriate.
