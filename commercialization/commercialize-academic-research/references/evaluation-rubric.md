# Evaluation Rubric

Use this rubric to compare commercialization paths consistently. A simple 1-5 scale is enough; the point is disciplined comparison, not fake precision.

## Dimensions

### Problem severity

- `1`: inconvenience, weak urgency, optional budget
- `3`: meaningful operational pain with some budget ownership
- `5`: acute pain tied to revenue, cost, risk, or compliance

### Proof burden

- `1`: long validation cycle, heavy evidence burden, many dependencies
- `3`: moderate pilot and integration work required
- `5`: buyers can test value quickly with limited friction

### Time to value

- `1`: value realization is distant or back-loaded
- `3`: value can be demonstrated in months
- `5`: value is visible in a pilot or short deployment

### Adoption friction

- `1`: major workflow change, procurement friction, or retraining
- `3`: manageable integration and process change
- `5`: drops into an existing workflow with minimal disruption

### Capital intensity

- `1`: large upfront funding, long build, or hard infrastructure needs
- `3`: moderate capital or hiring needs
- `5`: lean path to early traction

### Defensibility

- `1`: easy to copy, weak structural moat
- `3`: some differentiation through execution or data
- `5`: durable advantage through IP, workflow control, data, distribution, or switching cost

### Team fit

- `1`: path requires capabilities the current team does not have
- `3`: stretch but feasible with selective hires or partners
- `5`: strong match with the team's expertise, access, and credibility

## Decision heuristics

- Favor options with a credible path to learning and revenue, not just the largest upside.
- Penalize paths where the proof burden and channel burden compound each other.
- If two paths score similarly, prefer the one with faster reality-testing.
- If the top option depends on multiple unverified assumptions, downgrade it until the assumptions are tested.
